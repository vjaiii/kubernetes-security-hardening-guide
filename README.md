# Kubernetes Security Hardening Guide

This repository documents practical Kubernetes security guidance for securing clusters, workloads, access paths, and runtime behavior.

The goal is to capture reusable hardening ideas that help teams reduce risk in containerized platforms.

---

## Why this repository exists

Kubernetes is powerful, but it increases complexity across:

- identity and access control
- workload isolation
- secrets management
- network segmentation
- image trust
- runtime visibility

This repository collects practical guidance for building and reviewing more secure Kubernetes environments.

---

## Topics covered

- RBAC and least privilege access
- Workload identity
- Network policies
- Pod security standards
- Secrets handling
- Image security
- Admission controls
- Runtime monitoring
